
Clase Producto:
- El atributo precioCosto, no fue eliminado, ya que será utilizado para indicar el precio de costo actual
  del producto. Ese precio costo actual del producto se setea cuando se hace el pedido. Es decir si hago 
  un pedido de super a YPF, el precio de costo del producto será el que posee YPF para ese producto.

- En la tabla de la BD ProductoxProveedor, se agregó el atributo precioCostoProducto, que es el precio de costo que posee 
  ese producto para ese proveedor. 
  El DAO_ProveedorxProducto ya está adaptado al cambio, no hay que hacer nada en sus métodos, que son:
	* public boolean desmaterializarProveedorxProductos_CmdInsert(Proveedor proveedor, Lista productos):
	  ya inserta el atributo precio costo en la tabla ProductoxProveedor.
	* public boolean desmaterializarProveedorxProductos_CmdUpdate(Proveedor proveedor, Producto producto, String condicionUpdate)
 	  ya actualiza el atributo precio costo en la tabla ProductoxProveedor, hay que especificarle en la condicionUpdate
	  para que proveedor y producto (where fk_Producto=... and fk_Proveedor=...)
	* public Lista materializarProductosProveedor (Proveedor proveedor):
	  Devuelve una lista de productos del proveedor que se recibe por parámetro, cada uno con el precio de costo 
          del proveedor pasado por parámetro.

 
CU Registrar Proveedor:

- Para registar ahora un nuevo proveedor, en la plla de asignación de productos se debe seleccionar
  los productos y para cada uno ingresar el precioCosto. Luego se debe llamar el método del DAO_ProveedorxProducto
  "desmaterializarProveedorxProductos_CmdInsert(Proveedor proveedor, Lista productos)", y sigue todo como estaba antes.
  El único cambio es a nivel interfaz y el seteo del precio costo para cada producto seleccionado para el proveedor que se
  está por crear.
